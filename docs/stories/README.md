# Stories

Feature packets. One file per user-facing change or ported module:
`US-XXX-short-slug.md`, from `../templates/story.md`.

Numbering sequential. A story links the Android module/behavior it ports,
its acceptance criteria, and its validation proof.

**High-risk lane** (requires the Risk Criteria section filled): anything
touching user data integrity — backup import/export, library database
migration, download storage, the extension runtime.
